+### Using
+Gson using Java reflection API for direct setting field values. If there is a need to set your own field translation rules, you can use the following strategy:
+1. The model class must have the annotation @JsonSpecialRules. If the class does not have an annotation, it means that Gson is working in standard mode and using Java reflection API.
+```
+@JsonSpecialRules
+public class Model {
+
+}
+```
+2. Methods must be connected to the fields using the annotation @JsonConnectTo
+```
+@JsonSpecialRules
+public class Model {
+    private String name;
+    
+    @JsonConnectTo(targetField="name", rawName="firstName")
+    private void setName(String name) {
+        this.name = name;
+    }
+    
+    @JsonConnectTo(targetField="name", rawName="firstName")
+    private String getName() {
+        return this.name;
+    }
+}
+```
+`targetName` - the name of the target field.
+`rawName` (optional) - the raw name of the field in JSON body. If the attribute is not specified, the target field name is used.
